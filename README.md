# Texture-Hidden Anti-Counterfeiting QR Reproduction

This project reproduces the QR generation side of the paper "A Texture-Hidden Anti-Counterfeiting QR Code and Authentication Method".

The script generates:

- a standard QR code
- a refined QR code with reduced data code points
- Gaussian random texture
- bilinear-interpolated texture
- Floyd-Steinberg halftone texture
- the final texture-hidden QR code
- DFT spectrum visualizations and a simulated copied-code comparison

## Environment

The local reproduction was run with the `CDP1` Python environment:

```powershell
C:\Users\zpf\anaconda3\envs\CDP1\python.exe -m pip install -r requirements.txt
```

## Run

Paper-parameter reproduction. This uses a reduced code point size of `5 x 5`, Gaussian mean `mu = 120`, and QR version 3:

```powershell
C:\Users\zpf\anaconda3\envs\CDP1\python.exe main.py
```

Output directory:

```text
outputs/texture_qr_generation/
```

A software-decoder-friendly variant can be generated by increasing the reduced code point size and preserving QR timing patterns:

```powershell
C:\Users\zpf\anaconda3\envs\CDP1\python.exe main.py --point-px 11 --preserve-timing --output-dir outputs/texture_qr_generation_decode_friendly
```

The `11 x 11` plus `--preserve-timing` variant keeps the same generation pipeline but is easier for OpenCV's QR decoder to read from a purely digital image.

## Notes

The paper's `5 x 5` code point setting is selected for physical print and capture experiments. In a direct digital image, common software QR decoders may fail because the data modules are intentionally hidden in a strong random texture. The generated `5 x 5` version is therefore the closer visual reproduction, while the `11 x 11` version is useful for checking the encoded QR payload.
